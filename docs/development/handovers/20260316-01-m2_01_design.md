## Decisions made this session

| Decision | Rationale | Recorded in |
|---|---|---|
| Image naming: `<project>-agent-sandbox` (capability), `<project>-opencode-agent` (reasoning), hyphens | Docker Compose accepts hyphens natively; no underscore conversion needed | This handover → `tool_interface.md` |
| Tool interface spec: `docs/architecture/tool_interface.md`, architecture document | Defines external contract (command shapes, naming, mount guarantees); distinct from `execution_model.md` (internal implementation) | This handover → `project_index.md` |
| Docker Compose per project: `start_agent.sh` writes `docker-compose.yml` + `.env` into `SANDBOX_DIR` from template | Declarative container orchestration replaces imperative `docker run`; `.env` separates machine-specific from project-specific | This handover |
| Dogfooding first: agent-sandbox's own repo gets real compose file before template is produced | Template derived from working dogfood version, not the other way around | This handover |
| Compose baked vs `.env` split: image/container names, dependencies, volume, internal mounts baked; host paths, ports, credentials in `.env` | Stable project structure vs machine-specific runtime are different concerns | This handover |
| `.env` runtime variables: `SNAPSHOT_DIR`, `CHANGES_DIR`, `AGENT_INPUT_DIR`, `AGENT_OUTPUT_DIR`, `SERVE_PORT`, `OPENCODE_SERVER_PASSWORD` | Each maps to a host path or credential that varies per machine/run | This handover |
| Mode overrides: base `docker-compose.yml` has no ports; `docker-compose.serve.yml` adds ports; dry-run gets own override; `start_agent.sh` uses `-f` flags | Ports not exposed when not in serve mode; overrides compose cleanly | This handover |
| Build command: `make build [sandbox\|agent\|all]` dispatch pattern | Consistent arg passing; granular rebuild; staleness integrated per-image | This handover |
| Two-image staleness: both images get digest labels, separate `image-files.txt`, check both, warn separately, rebuild only stale | Images may go stale independently; operator needs to know which | This handover → `image.sh` |
| `start_agent.sh` remains entry point: gains `docker compose up/down`; preserves snapshot, checkpoint, validation | Compose handles container orchestration only; all other behaviour retained | This handover |
| Capability Dockerfile (`Dockerfile.sandbox`) lives in `SANDBOX_DIR`, project-controlled | Projects dockerize their own dev environment; capability layer is project-specific, not provider-specific | This handover |
| `image-files.txt` for capability: in `SANDBOX_DIR` alongside `Dockerfile.sandbox`; reasoning: stays in `providers/opencode/` | Each image's build inputs tracked where the Dockerfile lives | This handover → `image.sh` |
| Dry-run minimum guarantees: both containers start, reasoning accesses and writes to sandbox, graceful termination, diff written | Defines the contract for what a successful dry-run proves; existing checks to be catalogued and explicitly preserved or dropped | This handover → `tool_interface.md` |
| Capability Dockerfile: `dockerfile-default.sandbox` template in `libs/_template/`; copied to `SANDBOX_DIR` if project has no override; compose references the project-level file | Projects control their own dev environment; default template provides a working starting point; compose file is the integration point | This handover |
| Dry-run check catalogue and Dockerfile generation logic are implementation tasks, not open design questions | Minimum guarantees and generation pattern are decided; remaining work is inventory and code | This handover |

## Acceptance criteria

- `make start` brings up two containers via `docker compose up`; capability layer starts first (service dependency)
- Agent modifies a file in `sandbox/`; `staged.diff` appears in `SANDBOX_DIR/.workspace/changes/` after session ends
- `make apply` applies the diff cleanly to the host repo
- `make serve` invokes via `docker compose -f docker-compose.yml -f docker-compose.serve.yml up`; port exposed
- `make dry-run` invokes via `docker compose -f docker-compose.yml -f docker-compose.dry-run.yml up`; both containers start, reasoning writes to sandbox, graceful termination, diff written
- Capability layer container exits cleanly and triggers diff pipeline on `make stop` or natural session end
- Reasoning layer cannot see `SANDBOX_DIR/.snapshot/`; capability layer cannot see `SANDBOX_DIR/.workspace/agent-input/` or `SANDBOX_DIR/.workspace/agent-output/`
- `make build sandbox` builds only capability image; `make build agent` builds only reasoning image; `make build all` builds both
- Staleness check warns per-image; rebuild only rebuilds the stale image
