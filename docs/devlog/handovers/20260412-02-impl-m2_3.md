### Change 1 — Pre-session checkpoint tag (`scripts/start_agent.sh`)

Inserted after git validation, before workspace setup:

- Creates lightweight tag `agent-checkpoint/YYYYMMDD-HHMMSS` in `PROJECT_DIR` at session start
- Prunes `agent-checkpoint/*` tags to keep the 5 most recent (sorted by name, chronological)
- Writes the tag name to `$SANDBOX_DIR/.workspace/checkpoint-latest.ref` for use by `apply_workspace.sh`

### Change 2 — Format-patch + session-scoped artefact directory

**`scripts/start_agent.sh`:**
- Derives `SESSION_NAME` from sanitised branch name + checkpoint timestamp (e.g. `main-20260408-112344`)
- Exports `SESSION_NAME` so docker-compose can inject it into the container

**`libs/docker-compose.yml`:**
- Added `SESSION_NAME=${SESSION_NAME:-}` to the sandbox container environment block

**`libs/diff.sh`:**
- New function `diff_format_patch SANDBOX_DIR BASELINE_SHA PATCHES_DIR` — runs `git format-patch` to produce one numbered `.patch` file per agent commit; no-ops if there are no commits since baseline
- `diff_on_exit` updated: accepts optional 4th arg `SESSION_NAME`; writes artefacts under `CHANGES_DIR/<session-name>/staged.diff` and `CHANGES_DIR/<session-name>/patches/`; falls back to `CHANGES_DIR/` if `SESSION_NAME` is empty (backwards compatibility for older harness versions)
- `diff_on_autosave` updated: same `SESSION_NAME` 4th arg; writes `autosave.diff` into the session dir

**`scripts/sandbox-entrypoint.sh`:**
- EXIT trap and autosave loop both pass `${SESSION_NAME:-}` as the 4th argument to `diff_on_exit` / `diff_on_autosave`

### Change 3 — Apply workflow (`scripts/apply_workspace.sh`, `scripts/agent-sandbox.sh`, `libs/_templates/Makefile.template`)

**`scripts/apply_workspace.sh`** — complete rewrite:
- `draft [--session=<name>]`: resolves session dir (most recent if not specified); reads `checkpoint-latest.ref` to find the base tag; creates `agent/draft/<session-name>` branch from that tag; applies each `.patch` via `git am --3way` with per-patch author reset to operator's `git config user.name/email`; writes `.workspace/draft-state`
- `confirm [--target=<branch>]`: reads `draft-state`; rebases working branch onto target (defaults to `SOURCE_BRANCH`); fast-forward merges into target; deletes working branch; clears `draft-state`
- `reject`: reads `draft-state`; checks out `SOURCE_BRANCH`; deletes working branch; clears `draft-state`; session artefacts retained
- Legacy (no command, or `--mode=apply`): applies `staged.diff` with `git apply --3way`; no commits created; also searches most-recent session dir for `staged.diff` if root-level one not found

**`scripts/agent-sandbox.sh`:**
- Added `draft`, `confirm`, `reject` subcommands with `SESSION_ARG` and `TARGET_BRANCH` variables in the shared flag parser
- `apply` subcommand now hardcodes `--mode=apply` when calling `apply_workspace.sh`
- Error message for unknown subcommands updated to include new commands

**`libs/_templates/Makefile.template`:**
- Added `draft`, `confirm`, `reject` targets with `SESSION` and `TARGET_BRANCH` Make variables
- Operator workflow comment block added above the apply section
- `apply` target retained as legacy

### AC-1: Checkpoint tag creation

**Setup:** Run `agent-sandbox start` (or `make start`) against a project.

**AC-1.1** — Tag is created:
```bash
git -C "$PROJECT_DIR" tag --list 'agent-checkpoint/*'
# Expected: at least one tag of the form agent-checkpoint/YYYYMMDD-HHMMSS
```

**AC-1.2** — Ref file is written:
```bash
cat "$SANDBOX_DIR/.workspace/checkpoint-latest.ref"
# Expected: agent-checkpoint/YYYYMMDD-HHMMSS matching the tag above
```

**AC-1.3** — Pruning: run 6 sessions. After the 6th session:
```bash
git -C "$PROJECT_DIR" tag --list 'agent-checkpoint/*' | wc -l
# Expected: 5
```

### AC-2: SESSION_NAME and session-scoped artefacts

**Setup:** Run a session where the agent makes at least one commit. Let the container exit normally.

**AC-2.1** — Session directory is created:
```bash
ls "$SANDBOX_DIR/.workspace/changes/"
# Expected: one directory named <branch>-YYYYMMDD-HHMMSS
```

**AC-2.2** — `staged.diff` is written inside the session directory:
```bash
SESSION=$(ls "$SANDBOX_DIR/.workspace/changes/" | head -1)
ls "$SANDBOX_DIR/.workspace/changes/$SESSION/"
# Expected: staged.diff present (and non-empty if agent made changes)
```

**AC-2.3** — `patches/` directory is created and populated:
```bash
ls "$SANDBOX_DIR/.workspace/changes/$SESSION/patches/"
# Expected: one or more .patch files named 0001-....patch etc.
```

**AC-2.4** — Patch count matches agent commit count:
```bash
# Inside the container before exit (or inspect after), count agent commits
# Number of .patch files should equal number of commits since baseline
```

**AC-2.5** — No-change session produces no `staged.diff` and no patches:
```bash
# Run a session where the agent makes no changes.
ls "$SANDBOX_DIR/.workspace/changes/$SESSION/"
# Expected: patches/ directory exists but is empty; staged.diff absent
```

**AC-2.6** — Autosave writes to session directory:
```bash
# With AUTOSAVE_INTERVAL > 0, wait for first autosave interval.
ls "$SANDBOX_DIR/.workspace/changes/$SESSION/autosave.diff"
# Expected: file present and non-empty
```

**AC-2.7** — Multiple sessions accumulate without clobbering:
```bash
ls "$SANDBOX_DIR/.workspace/changes/"
# After two sessions: two separate directories, both intact
```

### AC-3: Draft workflow

**Setup:** A completed session with at least one commit. `checkpoint-latest.ref` and a `patches/` directory exist.

**AC-3.1** — `make draft` creates working branch and applies patches:
```bash
make draft  # or: agent-sandbox draft --project=... --sandbox=...
git -C "$PROJECT_DIR" branch --list 'agent/draft/*'
# Expected: agent/draft/<session-name> branch exists and is checked out
```

**AC-3.2** — Draft commits carry operator's author identity:
```bash
git -C "$PROJECT_DIR" log --format='%an <%ae>' HEAD~N..HEAD
# Expected: operator's git config name/email on all draft commits
```

**AC-3.3** — `draft-state` is written:
```bash
cat "$SANDBOX_DIR/.workspace/draft-state"
# Expected: SOURCE_BRANCH, WORKING_BRANCH, SESSION_DIR all set
```

**AC-3.4** — `make confirm` merges to source branch and cleans up:
```bash
make confirm
git -C "$PROJECT_DIR" branch --show-current
# Expected: back on SOURCE_BRANCH
git -C "$PROJECT_DIR" branch --list 'agent/draft/*'
# Expected: working branch deleted
git -C "$PROJECT_DIR" log --oneline -N
# Expected: draft commits now present on SOURCE_BRANCH, linear history
cat "$SANDBOX_DIR/.workspace/draft-state" 2>/dev/null
# Expected: file absent (cleared)
```

**AC-3.5** — `make confirm TARGET_BRANCH=other` merges to the specified branch:
```bash
git -C "$PROJECT_DIR" checkout -b other-branch
git -C "$PROJECT_DIR" checkout "$SOURCE_BRANCH"
make draft
make confirm TARGET_BRANCH=other-branch
git -C "$PROJECT_DIR" log other-branch --oneline -N
# Expected: draft commits on other-branch
```

**AC-3.6** — `make reject` discards draft and returns to source branch:
```bash
make draft
make reject
git -C "$PROJECT_DIR" branch --show-current
# Expected: SOURCE_BRANCH
git -C "$PROJECT_DIR" branch --list 'agent/draft/*'
# Expected: working branch deleted
cat "$SANDBOX_DIR/.workspace/draft-state" 2>/dev/null
# Expected: file absent
```

**AC-3.7** — Session artefacts are retained after both confirm and reject:
```bash
ls "$SANDBOX_DIR/.workspace/changes/$SESSION/patches/"
# Expected: .patch files still present
```

**AC-3.8** — Second `make draft` while one is in progress fails with clear error:
```bash
make draft
make draft  # should fail
# Expected: error message "a draft is already in progress"
```

## Test coverage

### `tests/test_diff.sh`
13 existing tests preserved. 15 new tests added:
- `diff_format_patch`: produces one patch per commit; correct `0001-` numbering; no-op on zero commits; fails on missing args; patches apply cleanly via `git am`
- `diff_on_exit` with SESSION_NAME: creates session dir; writes `staged.diff` and `patches/` inside it; sweep commit produces one patch; no patches on no-change session; does not leak to CHANGES_DIR root; multiple sessions accumulate as separate directories
- `diff_on_autosave` with SESSION_NAME: writes `autosave.diff` inside session dir; does not write to root
- Combined: exit and autosave write separate files inside the same session dir

### `tests/test_snapshot_host.sh`

- Copies tracked files; excludes gitignored files; includes untracked non-ignored files

- Handles unstaged move (old name absent, new name present)
- Excludes `.git/` directory; creates destination if absent; preserves nested directory structure; submodule pre-flight aborts correctly

### `tests/test_apply.sh`
Complete rewrite (old tests targeted deleted scripts). 34 new tests:
- **draft (9):** creates and checks out working branch; applies all patches; agent files present; author reset to operator identity; commit messages preserved; writes `draft-state`; most-recent session auto-selection; explicit `--session`
- **confirm (7):** returns to source branch; commits land on source; linear history (no merge commits); working branch deleted; `draft-state` cleared; `--target` branch; session artefacts retained
- **reject (5):** returns to source branch; working branch deleted; `draft-state` cleared; source HEAD unchanged; session artefacts retained
- **guards (7):** missing project; missing args; no `patches/` dir; empty `patches/`; double-draft blocked; confirm without draft; reject without draft
- **legacy `--mode=apply` (6):** applies diff to working tree; no commit created; fails on missing diff; fails on empty diff; falls back to session-dir `staged.diff` when no root-level diff exists
