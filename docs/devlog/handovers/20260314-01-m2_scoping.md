## Decisions made this session

| Decision | Rationale | Recorded in |
|---|---|---|
| Capability layer lives in project harness folder, not in `providers/` | Capability layer is project-specific; what it needs to support varies per project. `agent-sandbox` provides a minimal onboarding template only. | This handover; `execution_model.md` to be updated |
| `sandbox-entrypoint.sh` lives in `scripts/` | It is now general harness tooling, not provider-specific | This handover |
| No reasoning layer entrypoint for brief injection | `AGENTS.md` placed at `/home/agentuser/project/AGENTS.md` via Dockerfile; no entrypoint needed. Can be reintroduced if needed. | This handover |
| Reasoning layer working dir: `/home/agentuser/project/` | OpenCode scopes file operations to CWD; `sandbox/` and `.workspace/` must be visible as subdirectories. Home dir as CWD was rejected. | This handover |
| No CMD default in reasoning layer Dockerfile | `start_agent.sh` always supplies full invocation for all three modes (start, serve, dry-run) | This handover |
| `.agent-input/` retired as mount name | Superseded by `.snapshot/` (capability layer) and `.workspace/agent-input/` (reasoning layer) | This handover |
| Snapshot mounts as `SANDBOX_DIR/.snapshot/` — capability layer only (RO) | Snapshot is a harness internal; reasoning layer has no need to see it. `.snapshot/` is a clearer name than `.agent-input/`. | This handover |
| `.workspace/` split into three subdirectories | Separates concerns: `changes/` (capability layer diff output), `agent-input/` (reasoning layer task briefs, RO), `agent-output/` (reasoning layer progress/data, RW). Operator sees everything under one `.workspace/` tree. | This handover; `execution_model.md` and `agent_workflow.md` to be updated |
| Reasoning layer mounts `.workspace/agent-input/` at `/home/agentuser/project/.workspace/agent-input/` and `.workspace/agent-output/` at `/home/agentuser/project/.workspace/agent-output/` | Under CWD so agent can reach both without explicit path configuration | This handover |
| Reasoning layer is the less trusted party in capability↔reasoning boundary | Reasoning layer does the writes; capability layer is the exploited party whose diff pipeline has ripple effects to host repo | This handover; `security.md` to be updated |
| No binary files accepted in `.workspace/agent-output/` | Mitigation against reasoning layer writing malicious executable content through its output channel | This handover; `security.md` to be updated |

## Final mount shape

| Host path | Capability layer | Reasoning layer | Mode | Purpose |
|---|---|---|---|---|
| `SANDBOX_DIR/.snapshot/` | `/home/agentuser/.snapshot/` | — | RO | Project snapshot input |
| `SANDBOX_DIR/.workspace/agent-input/` | — | `/home/agentuser/project/.workspace/agent-input/` | RO | Task briefs, operator addenda |
| `SANDBOX_DIR/.workspace/agent-output/` | — | `/home/agentuser/project/.workspace/agent-output/` | RW | Agent progress, serialised data (no binaries) |
| `SANDBOX_DIR/.workspace/changes/` | `/home/agentuser/.workspace/changes/` | — | RW | Diff output (`staged.diff`, `autosave.diff`) |
| Shared Docker volume | `/home/agentuser/sandbox/` | `/home/agentuser/project/sandbox/` | RW | Working content |
