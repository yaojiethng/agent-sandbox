## Decisions made

**rsync for snapshot (Change 4):** `snapshot_enumerate_files` + `snapshot_copy_files` replaced by `snapshot_copy_worktree` using `rsync -a --filter=':- .gitignore' --exclude='.git'`. Copies working tree directly — not index-driven. Handles deletions, moves, new files correctly by construction. Known limitation: global gitignore (`~/.gitignore_global`, `~/.config/git/ignore`) and `.git/info/exclude` not respected. In practice these contain OS artifacts; operators relying on them for sensitive exclusions must duplicate patterns in local `.gitignore`. Documented, not a blocker.

**Session-scoped artefact directory (Change 2):** `staged.diff` moves from a fixed filename to `.workspace/changes/<session-name>/staged.diff`. Patches at `.workspace/changes/<session-name>/patches/`. `SESSION_NAME` derived at session start as `<sanitized-branch>-<timestamp>` (branch `/` replaced with `-`). Passed to container as env var. Multiple sessions accumulate without clobbering.

**Checkpoint tag retention (Change 1):** Keep last 5 `agent-checkpoint/*` tags. Prune on each new tag creation, sorted by tag name (chronological given `YYYYMMDD-HHMMSS` format).

**draft/confirm/reject workflow (Change 3):**
- `make draft` — applies patches to `agent/draft/<session-name>` branch via `git am` per-patch loop with author reset. Writes `.workspace/draft-state`.
- `make confirm [TARGET=<branch>]` — rebases draft branch onto target, then fast-forward merges. Linear history always. No merge commit unless there is a conflict requiring manual resolution. On conflict: `git rebase --continue`.
- `make reject` — discards draft branch, restores source branch, clears state.
- `make apply --mode=apply` retained as legacy fallback (old `git apply` behaviour).

**Author identity (Change 3):** Per-patch `git commit --amend --author="$(git config user.name) <$(git config user.email)>" --no-edit`. Resets name/email to operator's git config at `make draft` time. Commit message and author timestamp preserved from agent's patch. No `session.json` needed.

**Agent commit messages:** Agent writes its own commits. Messages applied as-is. Model attribution encouraged in messages but not enforced by the harness.
