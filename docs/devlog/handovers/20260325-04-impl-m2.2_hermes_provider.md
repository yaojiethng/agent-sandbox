## Acceptance criteria

- [ ] `agent-sandbox onboard` produces `.env` with stubs from all `providers/*/env.example`; no `docker-compose.serve.yml` in `SANDBOX_DIR`
- [ ] `make build PROVIDER=opencode` builds `opencode-agent-<project>` image
- [ ] `make build` (no PROVIDER) builds sandbox + all providers
- [ ] `make dry-run PROVIDER=opencode` passes — regression check on refactored infrastructure
- [ ] `make serve PROVIDER=opencode` resolves serve overlay from `providers/opencode/` not `SANDBOX_DIR`
- [ ] `make build PROVIDER=hermes` builds `hermes-agent-<project>` image
- [ ] `make dry-run PROVIDER=hermes` passes — Hermes liveness check
- [ ] A second provider can be added under `providers/<n>/` with `build.sh`, `run.sh`, `docker-compose.serve.yml`, `.env.example` — no changes to `scripts/` or `libs/` required
- [ ] Architecture documents in scope describe the system as built

## Decisions made this session

| Decision | Rationale | Recorded in |
|---|---|---|
| Claude Code provider: removed from roadmap | Investigation complete; verdict unappealing | `roadmap.md` |
| Hermes is the one additional provider for M2.2 | Pi and Claude Desktop remain informal future candidates | `roadmap.md` |
| Serve overlay moves to `providers/<n>/docker-compose.serve.yml` — static, not copied to `SANDBOX_DIR` | Provider-specific; no project-specific values; referenced by `run.sh` directly | This handover |
| `OPENCODE_SERVER_PASSWORD` in OpenCode serve overlay only | Only relevant for OpenCode serve mode; base compose stays clean | `providers/opencode/docker-compose.serve.yml` |
| Provider `.env` stubs from `providers/<n>/.env.example` — iterated by `onboard.sh` | Adding a provider requires no changes to `onboard.sh` | `onboard.sh` |
| `onboard.sh` has no `--provider` flag — onboarding is project setup, not provider selection | Provider selected at run time | `onboard.sh` |
| Provider selection via Make variables (`PROVIDER=hermes`) not trailing words | Idiomatic Make; no goals-string parsing; no intermediate dispatch script | `Makefile.template`, `agent-sandbox.sh` |
| `--rebuild` is a flag on run targets, not a separate subcommand | Scoped to containers required for the run; simpler AST | `agent-sandbox.sh`, `Makefile.template` |
| `build` iterates `providers/*/build.sh` glob — no hardcoded provider list | Adding a provider requires no changes to `agent-sandbox.sh` | `agent-sandbox.sh` |
| `make build-all`, `make build-sandbox`, `make build-agent` removed | Superseded by `make build` with optional `PROVIDER=` | `Makefile.template` |
